🔐 GitDB Token 混淆工具

将 GitHub Token 混淆成非明文格式，避免被 GitHub 扫描检测

⚠️ 重要提示：

这只是一个简单的字符串混淆，不是加密！目的是防止 GitHub 自动扫描检测到明文 token。

混淆后的 token 格式：gitdb_xxxxxxxxxxxx

🔒 Token 混淆 生成混淆 Token

获取 Token: GitHub Settings

🔓 Token 解混淆 还原原始 Token

📖 使用说明

步骤 1: 生成混淆 Token

  1. 在上方输入你的 GitHub Token
  2. 点击"生成混淆 Token"
  3. 复制生成的混淆 Token（格式：gitdb_xxx）

步骤 2: 配置到项目

将混淆后的 Token 配置到你的项目中：

// 方式 1: 直接配置 const db = new GitDB({ owner: 'DarrenHost', repo: 'gitdb', token: 'gitdb_xxxxxxxxxxxx' // 混淆后的 token }); // 方式 2: 环境变量 window.GITDB_TOKEN = 'gitdb_xxxxxxxxxxxx';

步骤 3: 自动解混淆

GitDB 会自动检测并解混淆 token，无需手动处理！

🔐 安全提示

  • 混淆后的 token 仍然具有完整权限，请妥善保管
  • 不要将混淆工具页面部署到公开环境
  • 定期更换 token，建议每 3-6 个月
  • 使用 Fine-grained Token，只授予必要权限
  • 如果 token 泄露，立即在 GitHub 撤销
